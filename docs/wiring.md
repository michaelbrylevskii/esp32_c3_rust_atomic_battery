# Подключение оборудования

Документ описывает подключение текущего binary target `app` для платы
`ESP32-C3 Super Mini`.

## Состав оборудования

| Компонент | Роль |
| --- | --- |
| `ESP32-C3 Super Mini` | основной контроллер |
| `PN532` | NFC reader в режиме I2C |
| `TM1637` | 4-разрядный семисегментный дисплей с двоеточием |
| Красный LED | индикация ошибки, ожидания или запрета запуска |
| Зелёный LED | индикация активной работы |
| Тумблер | вход активации устройства |

## Таблица подключений

| Компонент | Сигнал | ESP32-C3 | Электрическая логика |
| --- | --- | --- | --- |
| `PN532` | `SDA` | `GPIO3` | I2C data |
| `PN532` | `SCL` | `GPIO4` | I2C clock |
| `PN532` | `VCC` | `3.3V` | питание модуля |
| `PN532` | `GND` | `GND` | общая земля |
| `PN532` | `RSTO` / `RSTPD_N` | не подключён | reset pin зарезервирован на будущее |
| `TM1637` | `CLK` | `GPIO5` | линия протокола TM1637 |
| `TM1637` | `DIO` | `GPIO6` | линия протокола TM1637 |
| `TM1637` | `VCC` | `3.3V` | питание модуля |
| `TM1637` | `GND` | `GND` | общая земля |
| Красный LED | `anode` | `GPIO0` через резистор | active-high output |
| Красный LED | `cathode` | `GND` | общий минус |
| Зелёный LED | `anode` | `GPIO1` через резистор | active-high output |
| Зелёный LED | `cathode` | `GND` | общий минус |
| Тумблер | вход | `GPIO10` | input с внутренним pull-up |
| Тумблер | второй контакт | `GND` | замыкание на землю включает устройство |
| On-board LED | встроенный LED | `GPIO8` | обычно active-low, внешний провод не нужен |

## PN532 и I2C

PN532 должен быть переведён в I2C mode аппаратно: джамперами, DIP-switch или
перемычками на конкретном модуле.

Для линий I2C нужны внешние pull-up резисторы:

| Линия | Pull-up |
| --- | --- |
| `SDA` | `4.7k` к `3.3V` |
| `SCL` | `4.7k` к `3.3V` |

Внутренние pull-up ESP32-C3 слишком слабые для стабильного I2C. Если PN532
нестабильно читает firmware version, периодически теряет метку или зависает
после reset ESP32, сначала проверь питание модуля, общую землю и внешние
pull-up.

Reset pin PN532 сейчас не подключён. Если модуль продолжит зависать после reset
ESP32, стоит подключить reset pin к свободному GPIO и добавить программный reset
в инициализацию NFC.

## LED

Внешние LED подключаются как active-high output:

```text
GPIO -> токоограничивающий резистор -> LED anode
LED cathode -> GND
```

Токоограничивающий резистор обязателен. Для макетной проверки обычно подходит
диапазон `220..1000 Ohm`.

`set_drive_strength(...)` на GPIO не заменяет токоограничивающий резистор. Это
настройка выходного драйвера пина, а не ограничение тока через LED.

## Тумблер

Тумблер подключается между `GPIO10` и `GND`. В прошивке вход настроен как
`Pull::Up`.

| Уровень на `GPIO10` | Состояние приложения |
| --- | --- |
| `High` | тумблер выключен |
| `Low` | тумблер включён |

## Пины риска на ESP32-C3 Super Mini

| GPIO | Причина |
| --- | --- |
| `GPIO2` | strapping pin |
| `GPIO8` | strapping pin, часто on-board LED |
| `GPIO9` | strapping pin, часто BOOT |
| `GPIO12..17` | обычно линии SPI flash |
| `GPIO18/19` | USB D-/D+ |
| `GPIO20/21` | UART0 |
| `GPIO4..7` | могут пересекаться с JTAG |

Текущая макетная схема использует `GPIO4..7` для PN532/TM1637. Это нормально
для обычной разработки, но при аппаратной JTAG-отладке эти сигналы лучше
перенести на другие GPIO.

## Инструменты для схем

В репозитории пока нет breadboard-схемы или принципиальной схемы в CAD-формате.
Для дальнейшей документации подойдут:

| Инструмент | Когда использовать |
| --- | --- |
| `Fritzing` | breadboard-вид, понятный для сборки на макетной плате |
| `KiCad` | принципиальная схема и дальнейшая PCB |
| `Wokwi` | симуляция, если хватает моделей нужных модулей |
